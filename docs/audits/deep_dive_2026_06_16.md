# Deep-Dive Audit — 2026-06-16 (v9.419)

6-dimension multi-agent deep dive (correctness, data-integrity, hooks/realtime,
security, sport-science, robustness) + adversarial verification. ~25 verified
findings. This file records every finding and its disposition — FIXED in v9.419,
or DEFERRED with reason. Fixed items also carry regression tests.

## FIXED in v9.419

### Correctness (pure libs)
- **CRITICAL — raceGoalProjection.js `projectCTLAtRace`**: `dailyTSS * TAU * (1-decay)`
  had a spurious `×42`; a daily EWMA converges to `dailyTSS`, not `dailyTSS×42`.
  Projected CTL was inflated ~42× → `assessRaceReadiness` reported `on_track` for
  essentially everyone. Fixed to `currentCTL*decay + dailyTSS*(1-decay)` + magnitude test.
- **CRITICAL/HIGH — `durationMin` field mismatch (6 libs)**: `sanitizeLogEntry`
  stores minutes as `duration` (never `durationMin`), so longRunConsistency,
  longestSessionTrend, longRunFrequency, paceByRpe, paceRange, lifetimeTotals read
  `undefined` for every real entry and silently self-hid. Added a `duration`
  fallback (mirrors cyclingNpTrend). Tests only passed before because they injected
  `durationMin`, bypassing the sanitizer.
- **HIGH — injuryForecast.js**: history chart sliced `log` causally but passed the
  FULL `recovery` array into past points (future data leak). Now slices recovery too.
- **HIGH — runningCV.js**: required `distanceM`, dropping manual runs stored as
  `distanceKm`. Now normalizes distanceM/distanceKm/distance.
- **MED — rowing.js**: `strokeEfficiency`/`rowingEfficiencyFactor` didn't guard
  missing `distanceM` → NaN (truthy) → RowingMetricsCard rendered literal "NaN".
  Now return null.
- **MED — trainingMonotonyStrain.js**: used sample stdev `/(n-1)`; Foster (2001) and
  the app's own `computeMonotony` use population stdev `/n`. ~+8% inflation fixed.
- **MED — cpDecay.js**: detraining slope fit over full history while decayPct used a
  12-week window → recent decline hidden for long-history athletes. Slope now uses
  the same 12-week window.
- **LOW — neuromuscularFreshness.js**: off-by-one windows (8-day "7d", 29-day "28d").
- **LOW — intelligence.js getSingleSuggestion**: inline ACWR replaced with canonical
  `calculateACWR` (cross-surface consistency).
- **LOW — goalTracker.js**: progress pct now floored at 0 (was negative for regress).
- **LOW — rowing.js**: t-critical hardcoded df=4 → df-keyed t-table.
- **LOW — afterBigWeekRpe.js**: PROLONGED_ELEVATION no longer fires when RPE fell.

### Data integrity / storage
- **HIGH — storage.js exportAllData**: omitted `sporeus-injuries`,
  `sporeus-race-results`, `sporeus-training-age` (in SCHEMA gap) → "Download Backup"
  silently dropped them. Now enumerates all `sporeus`-prefixed keys (future-proof).
- **MED — storage.js importAllData**: non-atomic + returned success on malformed
  (`"5"`, `[1,2]`) or partial-write (quota) input → reload into corruption. Now
  validates object shape and returns false if any key write fails.
- **MED — offlineQueue.js flushQueue**: counted only `failed`, not `skipped`, so
  poison writes (≥3 attempts) showed "synced" while dead. Now `skipped>0` ⇒ not synced.

### Hooks / realtime
- **MED — useRealtimeSquad.js**: untracked 6s toast `setTimeout` → setState-after-
  unmount + stacking. Now tracked/cleared. Added MAX_RETRY=8 reconnect cap +
  channel release (was perpetual orphan); clean CLOSED no longer auto-reconnects.
- **LOW — useSessionAttendance.js**: same uncapped reconnect → MAX_RETRY=8 cap.

### Robustness
- **MED — UploadActivity.jsx**: parse stage could hang the spinner forever if the
  edge fn returned 200 then died (no terminal status). Added a 90s watchdog →
  error+retry. Guarded the realtime handler against an absent `new` payload.

### Security
- **MED — comment-notification edge fn**: was `verify_jwt=false` with NO inbound
  gate — any anonymous caller triggered its service-role queries + send-push relay
  (push spam). Added `isVerifiedServiceCall`. **DEPLOY ORDER (fail-closed):** the
  `session_comments` INSERT DB webhook must send `x-sporeus-webhook-secret:
  <WEBHOOK_SECRET>` BEFORE this fn is deployed, or comment notifications stop.
  (Code in repo; operator deploy-gated, per the established patch-webhook-first pattern.)

## DEFERRED (with reason)

- **coach_invites RLS SELECT leak (MED)** — policy `USING (auth.uid()=coach_id OR
  (used_by IS NULL AND expires_at>now()))` lets any authenticated user enumerate all
  unused invite `code`s and redeem into any coach's roster. **NOT a naive policy
  drop:** the athlete-side redemption (`MyCoach.jsx JoinCoachInput.lookup/accept`)
  reads `coach_invites` BY CODE client-side and relies on that OR branch — dropping it
  breaks coach↔athlete linking. Proper fix = a `lookup_coach_invite(code)` SECURITY
  DEFINER RPC (keyed by the code the user already holds, no enumeration) + move
  redemption server-side, THEN restrict SELECT to `coach_id=auth.uid()`. Table is
  EMPTY in prod (0 rows) → not currently exploitable. Deferred to a focused change.
- **intelligence calcLoad EWMA constants (HIGH-consistency)** — `calcLoad` uses
  `2/(N+1)` (ATL 0.25 / CTL 0.0465) while the canonical PMC engine `calculatePMC`
  uses `1−e^(−1/N)` (0.133 / 0.0235); the intelligence.js comment falsely claims they
  match. Steady-state agrees; transient TSB diverges across the app. Engine-level
  change with broad ripple (re-baselines many tests, shifts user-facing readiness) +
  founder owns sport-science modeling → deferred for a founder decision (align the
  engine vs. correct the comment).
- **Hydration-vs-mutation race (MED)** — `useSyncedTable`/`useRecovery` (older
  pattern): a write during the hydration window is shown locally but the DB write is
  skipped, then hydrate replaces local state with server rows → the just-entered row
  can vanish on slow cold logins. Narrow window; fix needs merge-not-replace or
  queued mutation. Deferred (involved; the newer `useTrainingLogQuery` path is safe).
- **Strava OAuth static `state:'strava'` (LOW CSRF)** — no per-session nonce. App.jsx
  OAuth routing disambiguates Supabase vs Strava by PRESENCE of `state`; changing the
  value risks breaking auth routing for a limited-impact CSRF (attacker links own
  Strava to a victim). Deferred — needs care in App.jsx + sessionStorage nonce.
- **attribution-log unverified JWT (LOW)** — derives `userId` from a base64-decoded
  (unverified) JWT to stamp `profiles.first_touch`. Edge fn + operator deploy; only a
  null non-sensitive marketing field. Fix: `getUser()` instead of decode. Deferred.
- **deloadDetector.js baseline (LOW)** — neighbor average uses all weeks vs
  surrounding weeks. Changes outputs; low value. Deferred.
- **encrypt_device_token anon-executable (LOW)** — DEFINER encryption oracle; leaks
  no key/plaintext, no matching anon decrypt. Hygiene-only; revoking risks device
  sync. Deferred.
