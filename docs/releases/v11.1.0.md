# Sporeus Athlete App — v11.1.0 Release Notes

**Released:** 2026-04-23  
**Type:** Security & Compliance hardening  
**Previous version:** v11.0.10

---

## Phase 1 Complete: Webhook → Rights → Security

v11.1.0 delivers three sequential blocks that harden the platform end-to-end.

---

## Block 1: Webhook State Machine (v11.0.10 → v11.1.0)

Replaced the ad-hoc webhook handler with a fully auditable subscription state machine.

- `apply_subscription_event(p_event jsonb)` SQL function — single-function state machine; every Dodo + Stripe event routed through one auditable path
- `processed_webhooks` deduplication table — HMAC-verified idempotency key; replays safe
- `dodo-webhook` edge function rewritten — delegates all state transitions to SQL; email side-effects only
- `apply_tier_change()` — atomic tier + billing_event write in one transaction
- `subscription_hardening` migration — grace period enforcement, expiry cascade, status machine
- `subscription_events` audit table — immutable log of every state transition

---

## Block 2: KVKK / GDPR Data Rights (v11.1.0 → v11.1.1)

Full data portability and deletion with 30-day grace period.

- `data_rights_requests` table — unified export + deletion request tracking
- `build_user_export()` — 23-table JSON export (excludes device token + embeddings)
- `purge_user()` — FK-safe cascade: archives coach_sessions, soft-deletes comments, purges 24 tables
- `export-user-data` edge function — authenticated; uploads to user-exports bucket; returns 7-day signed URL
- `purge-deleted-accounts` edge function — daily cron; processes past-grace-period deletions; sends confirmation email
- `DataPrivacySettings.jsx` — in-app export + deletion UI with two-step confirmation and cancellation
- Legal docs: `docs/legal/kvkk-notice-tr.md` + `docs/legal/privacy-notice-en.md`
- Ops: `docs/ops/runbooks.md` — deletion inquiry runbook, immediate purge SQL

## Block 3: Security Hardening (this block)

### SECURITY DEFINER search_path

All 27 SECURITY DEFINER functions now have `SET search_path = ''`:
- 17 functions: `ALTER FUNCTION ... SET search_path = ''` (bodies already qualified)
- 10 functions: bodies rewritten to use `public.*` table names before setting `search_path = ''`

Functions fixed: `ack_ai_session_msg`, `decrypt_device_token`, `delete_ai_batch_msg`, `delete_push_fanout_msg`, `delete_strava_backfill_msg`, `drain_ai_session_queue`, `encrypt_device_token`, `enqueue_ai_batch`, `enqueue_push_fanout`, `enqueue_session_analysis`, `enqueue_strava_backfill`, `inject_tier_jwt_claim`, `move_to_dlq`, `read_ai_batch`, `read_push_fanout`, `read_strava_backfill`, `refresh_queue_metrics`, `get_my_tier`, `get_load_timeline`, `get_squad_readiness`, `get_weekly_summary`, `refresh_mv_load`, `handle_new_user`, `increment_referral_uses`, `apply_tier_change`, `search_everything`, `get_squad_overview`

### RLS initplan fix

59 RLS policies updated: bare `auth.uid()` → `(SELECT auth.uid())`. PostgreSQL evaluates the subselect form once per query (initplan) rather than once per row, significantly improving query performance on large tables.

### Permissive policy consolidation

| Table | Before | After |
|-------|--------|-------|
| `coach_notes` | 3 permissive policies | 1 (ALL: coach writes, athlete reads) |
| `coach_sessions` SELECT | 2 SELECT policies + 1 ALL | 1 SELECT (merged) + 1 ALL |
| `messages` INSERT | 2 INSERT | 1 merged INSERT |
| `messages` SELECT | 2 SELECT | 1 merged SELECT |

### Index cleanup

- 26 unused indexes dropped (flagged by Supabase performance advisor)
- 3 missing FK indexes added: `activity_upload_jobs(log_entry_id)`, `coach_invites(used_by)`, `coach_notes(athlete_id)`
